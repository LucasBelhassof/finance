API de contas, cartoes e categorias

Gerencia contas bancarias, contas em caixa, cartoes de credito e categorias personalizadas do usuario. Essas entidades alimentam transacoes, faturas, dashboard, importacao e habitacao.

Escopo por usuarioCartao depende de conta paiCategorias de sistema protegidasBloqueios de exclusao
Capaz de fazerCriar, listar, editar e excluir contas e categorias; ajustar limite e fechamento de cartao.
LimitacoesCartao exige conta pai bancaria; categorias em uso ou de sistema nao podem ser removidas.
PermissoesBearer obrigatorio e filtros por user_id.

Visao operacional

Este dominio define onde o dinheiro entra ou sai e como as transacoes sao classificadas. Toda operacao financeira depende dessas estruturas, por isso o backend aplica validacoes de tipagem, exclusao segura e coerencia entre conta pai e cartao.

Capaz de fazer

  • Criar conta bancaria, caixa ou cartao de credito.
  • Listar conexoes bancarias/cartoes do usuario.
  • Editar nome, saldo, limite, dias de fechamento e vencimento.
  • Criar categoria customizada por tipo.
  • Listar categorias do usuario combinadas com categorias padrao do sistema.
  • Excluir categorias e contas apenas quando nao quebram relacionamentos existentes.

Nao faz

  • Nao permite remover a ultima conta de caixa.
  • Nao permite criar cartao sem parentBankConnectionId valido.
  • Nao permite enviar campos exclusivos de cartao para uma conta comum.

Limitacoes

  • Conta exige name, accountType e currentBalance.
  • Cartao exige tambem creditLimit, statementCloseDay, statementDueDay e conta pai bancaria.
  • Delete de conta bloqueia se houver cartoes filhos ou transacoes ligadas.
  • Delete de categoria bloqueia se houver uso em transacoes, habitacao, parcelamentos, regras de categorizacao, planos, drafts ou recomendacoes.

Endpoints relacionados

Metodo Rota Capacidade Auth Permissao Entrada Resposta Limitacao
GET /api/banks Lista contas e cartoes Bearer Usuario autenticado Sem body { banks } Inclui contas e cartoes do proprio usuario
POST /api/banks Cria conta/cartao Bearer Usuario autenticado Payload da conta 201 + conta Cartao exige pai bancario
PATCH /api/banks/:id Edita conta/cartao Bearer Usuario autenticado Payload parcial valido conta atualizada ID inteiro obrigatorio
DELETE /api/banks/:id Remove conta/cartao Bearer Usuario autenticado ID 204 Bloqueia recursos vinculados
GET /api/categories Lista categorias Bearer Usuario autenticado Sem body { categories } Mistura padrao + usuario
POST /api/categories Cria categoria Bearer Usuario autenticado name, type 201 + categoria Nome obrigatorio e tipo valido
PATCH /api/categories/:id Edita categoria Bearer Usuario autenticado name, type opcional categoria atualizada Categoria sistema nao deve ser quebrada
DELETE /api/categories/:id Exclui categoria Bearer Usuario autenticado ID 204 Bloqueia categoria em uso

Entradas esperadas

Campo Tipo Obrigatorio Regra Observacao
name string Sim Nao vazio Conta e categoria usam trim
accountType string Sim cash, bank_account ou credit_card Define campos obrigatorios
currentBalance number Sim Numero finito Saldo exibido no dashboard
creditLimit number Condicional Obrigatorio para cartao Nao deve ser enviado em conta comum
statementCloseDay / statementDueDay number Condicional 1 a 31 Usado por invoices
type string Sim para categoria income ou expense Define compatibilidade com transacoes

Validacoes

Validacao Quando ocorre Resultado Erro esperado
Conta pai bancaria Create/update de cartao Impede cartao sem conta pai adequada 400 invalid_parent_bank_connection
Ultima conta caixa Delete Impede exclusao do ultimo caixa 400 cannot_delete_last_cash_account
Categoria de sistema ou em uso Delete categoria Bloqueia exclusao 400 category_in_use / system_category_locked
Campos de cartao em conta comum Create/update Rejeita payload inconsistente 400 invalid_bank_payload

Efeitos colaterais

  • Cria/altera/remove registros em bank_connections e categories.
  • Afeta escolhas disponiveis em transacoes, importacao, invoices, dashboard e habitacao.
  • Invalida caches de bancos, dashboard, invoices e transacoes no frontend.

Erros esperados

HTTP Causa Quando Como corrigir
400 Payload inconsistente Create/update Rever tipo da conta e campos obrigatorios
404 Conta ou categoria inexistente Update/delete fora do escopo Usar recurso valido
401 Bearer ausente Qualquer rota Autenticar
