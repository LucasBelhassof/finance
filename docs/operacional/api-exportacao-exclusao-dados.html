API de exportacao e exclusao de dados

Permite baixar CSV de transacoes, export completo em JSON e excluir a conta do usuario autenticado mediante senha atual. A exclusao limpa o cookie de refresh no fim da operacao.

Export CSV e JSONExclusao destrutivaSenha atual obrigatoriaCookie limpo apos delete
Capaz de fazerGerar anexos baixaveis e excluir a conta inteira do usuario.
LimitacoesExport CSV cobre transacoes; export JSON cobre um conjunto maior de tabelas do usuario.
PermissoesBearer obrigatorio em todas as rotas e senha atual na exclusao.
Arquivos relacionadosserver/modules/user-data/*, src/hooks/use-user-data.ts, src/pages/Settings.tsx.

Visao operacional

O modulo atende operacoes de portabilidade e encerramento da conta. A exportacao prioriza seguranca do dado entregue e a exclusao passa por verificacao de senha antes de apagar o usuario e limpar a sessao web.

O CSV faz escaping de valores que poderiam ser interpretados como formula por planilhas, prefixando campos iniciados por =, +, -, @, tab ou carriage return.

Capaz de fazer

  • Baixar CSV de transacoes do usuario autenticado.
  • Baixar JSON completo com perfil e tabelas relacionadas.
  • Excluir a conta apos validar a senha atual.
  • Limpar refresh cookie ao concluir a exclusao.
  • Registrar auditoria de exclusao de conta.

Nao faz

  • Nao permite exclusao sem senha atual.
  • Nao exporta dados de outros usuarios.
  • Nao mantem sessao ativa apos exclusao da conta.

Limitacoes

  • CSV e focado em transacoes; dados de outras tabelas exigem JSON.
  • O conjunto exato do JSON depende do repositorio server/modules/user-data/repository.ts.
  • Retencao posterior ou backups fora da base transacional nao foram identificados no repositorio analisado.

Endpoints relacionados

Metodo Rota Capacidade Auth Permissao Entrada Resposta Limitacao
GET /api/user-data/export/csv Baixa CSV Bearer Usuario autenticado Sem body Anexo CSV Conteudo focado em transacoes
GET /api/user-data/export/json Baixa export completo Bearer Usuario autenticado Sem body Anexo JSON Sem selecao parcial por rota
DELETE /api/user-data/account Exclui conta Bearer Usuario autenticado { currentPassword } { message } + cookie limpo Operacao destrutiva e irreversivel na base ativa

Entradas esperadas

Campo Tipo Obrigatorio Regra Observacao
currentPassword string Sim para delete Passa pelo schema de exclusao Verificada com Argon2 no service
Headers de auth Bearer Sim Sem bearer as rotas nao executam Refresh cookie nao substitui o bearer

Validacoes

Validacao Quando ocorre Resultado Erro esperado
Bearer obrigatorio Todas as rotas Bloqueia export/delete anonimo 401
Senha atual correta DELETE account Impede exclusao sem credencial correta 403 invalid_credentials
Escaping de CSV GET export/csv Neutraliza risco de formula injection Nao e erro; e sanitizacao de saida

Efeitos colaterais

  • CSV e JSON fazem leitura, sem mutacao.
  • Delete account remove dados do usuario e gera auditoria account_deleted.
  • Delete limpa o cookie de refresh com opcoes expiradas.

Erros esperados

HTTP Causa Quando Como corrigir
401 Bearer ausente Export e delete Autenticar
403 Senha atual incorreta Delete account Informar senha correta
500 Falha ao montar export ou excluir dados Leitura/escrita da base Inspecionar requestId e logs

Checklist da capacidade

  • Se adicionar nova tabela sensivel, decidir se entra no JSON export.
  • Manter escaping do CSV contra formula injection.
  • Nunca remover verificacao de senha da exclusao de conta.
