API de dashboard e metricas

Consolida cards de saldo/receitas/despesas, gastos por categoria, contas, transacoes recentes e indicadores usados pelas telas Index.tsx e ExpenseMetrics.tsx.

Resumo financeiroFiltros por periodoCards e graficosInsights premium desabilitados
Capaz de fazerListar resumo do dashboard, contas, gastos por grupo e ultimas transacoes, com ou sem recorte de datas.
LimitacoesDepende de dados financeiros consistentes; /api/insights retorna lista vazia no codigo atual.
PermissoesTodas as rotas exigem bearer; insights e chat no dashboard sofrem sanitizacao para usuario free.

Visao operacional

O dashboard e a camada de leitura agregada do produto. Ele nao substitui o CRUD; consome transacoes, contas e categorias para montar cards, listas e graficos. O frontend ainda recalcula alguns cortes client-side para filtros mais detalhados em Index.tsx e ExpenseMetrics.tsx.

Capaz de fazer

  • Retornar cards de saldo, receitas e despesas.
  • Listar contas/cartoes do usuario.
  • Listar transacoes recentes.
  • Agregar gastos por grupo de categoria.
  • Aplicar intervalo startDate/endDate em /api/dashboard.
  • Servir base para charts de spending e metricas mais detalhadas.

Nao faz

  • Nao corrige automaticamente dados inconsistentes de transacao.
  • Nao retorna insights premium reais no estado atual; backend devolve array vazio e a tela de insights esta desabilitada.

Limitacoes

  • Sem dados financeiros, cards e listas ficam vazios ou zerados.
  • Filtros invalidos dependem de normalizacao em normalizeDashboardFilters.
  • Usuarios free recebem dashboard sanitizado, sem insights e sem chatMessages.

Efeitos colaterais

  • Nao cria registro por si so; e uma camada de leitura.
  • Recebe impacto imediato de mudancas em transacoes, contas, importacao e parcelamentos.
  • Tem cache React Query invalido por mutations de transacoes, contas e imports.
