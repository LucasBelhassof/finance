API de transacoes

Controla listagem, criacao, edicao e remocao de transacoes financeiras do usuario autenticado. Tambem participa da atualizacao de planos, dashboard, faturas e parcelamentos refletidos no frontend via React Query.

Escopo por usuarioCRUD principal do financeiroAfeta dashboard e planosIntegra parcelamentos
Capaz de fazerCriar receita ou despesa, listar com paginação/limit, editar transacao simples ou parcela, remover transacao simples ou truncar recorrencia.
LimitacoesReceita nao pode usar cartao de credito; categoria deve combinar com o tipo; payload minimo e obrigatorio.
PermissoesTodas as rotas exigem bearer valido e sempre filtram por user_id.
Arquivos relacionadosserver/app.ts, server/database.js, src/hooks/use-transactions.ts, src/pages/Transactions.tsx.

Visao operacional

Esta API resolve o registro diario do financeiro. Toda transacao precisa apontar para conta/cartao e categoria coerentes. Alteracoes podem disparar reavaliacao de planos e invalidacao de cache no frontend, e transacoes parceladas usam tratamento diferenciado para atualizar grupos inteiros.

Capaz de fazer

  • Criar transacao com descricao, valor, data, conta/cartao e categoria.
  • Listar transacoes recentes com limit ou paginação por page/pageSize.
  • Projetar receitas recorrentes positivas em certas visoes derivadas.
  • Editar parcela atual, todas, futuras, passadas ou conjunto customizado de uma compra parcelada.
  • Truncar serie recorrente ao excluir ou editar data futura de uma recorrencia.
  • Reavaliar planos afetados em create, update, delete e import.

Nao faz

  • Nao permite receita associada a cartao de credito.
  • Nao aceita categoria de tipo diferente do sinal do valor.
  • Nao expoe rotas sem autenticacao.

Limitacoes

  • description, amount, occurredOn e bankConnectionId sao obrigatorios.
  • Se categoryId faltar em despesa, usa categoria padrao compras; em receita, a categoria passa a ser obrigatoria.
  • Edicao de parcelamento depende de installmentUpdateScope e dos numeros selecionados no frontend.
  • Delete de recorrencia pode encerrar a serie em vez de apagar o historico inteiro.

Endpoints relacionados

Metodo Rota Capacidade Auth Permissao Entrada Resposta Limitacao
GET /api/transactions Lista transacoes Bearer Usuario autenticado limit ou page/pageSize { transactions } ou lista paginada Sem filtros server-side adicionais na rota
POST /api/transactions Cria transacao Bearer Usuario autenticado descricao, valor, data, conta, categoria 201 + transacao Receita nao usa cartao
PATCH /api/transactions/:id Edita transacao ou parcela Bearer Usuario autenticado Mesmo payload base + escopo de parcela transacao atualizada ID precisa ser inteiro valido
DELETE /api/transactions/:id Remove transacao ou trunca recorrencia Bearer Usuario autenticado ID na rota, body opcional com occurredOn 204 Recorrencia pode ser truncada

Entradas esperadas

Campo Tipo Obrigatorio Regra Observacao
description string Sim Nao vazia Trim no backend
amount number Sim Finito; sinal define income/expense Frontend edita valor absoluto e tipo separado
occurredOn data ISO curta Sim Normalizada para YYYY-MM-DD Tambem usada em recorrencia
bankConnectionId number Sim Inteiro existente do proprio usuario Escopo por user_id
categoryId number Condicional Obrigatoria para receita Despesa pode cair em categoria padrao
isRecurring boolean Nao So permanece true para valores positivos Recorrencia de despesa nao e mantida nesta regra

Validacoes

Validacao Quando ocorre Resultado Erro esperado
Campos obrigatorios Create e update Rejeita payload incompleto 400 invalid_transaction_payload
Conta/cartao existente Create, update e import Rejeita recurso de outro usuario ou inexistente 404 bank_connection_not_found
Conta valida para receita Create e update Impede receita em cartao 400 invalid_transaction_bank_connection
Categoria coerente com sinal Create, update e import Rejeita categoria de tipo incorreto 400 invalid_category
ID de rota inteiro PATCH e DELETE Bloqueia rota com ID invalido 400 invalid_transaction_id

Efeitos colaterais

  • Cria, altera ou remove registros em transactions.
  • Pode atualizar installment_purchases ao editar bundles parcelados.
  • Dispara reevaluateAffectedPlansForTransactions().
  • Invalida caches de transacoes, dashboard, planos, invoices, insights e overview de parcelamentos no frontend.

Erros esperados

HTTP Causa provavel Quando acontece Como corrigir
400 Payload invalido ou categoria incoerente Create/update Corrigir descricao, valor, data, conta e categoria
404 Transacao ou conta nao encontrada ID inexistente ou fora do escopo do usuario Usar recurso valido do proprio usuario
401 Bearer ausente Qualquer rota Autenticar novamente
