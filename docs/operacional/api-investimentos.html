API de investimentos

Gerencia caixas/objetivos de investimento do usuario, com modo de contribuicao fixa ou percentual de renda, valor atual, alvo e vinculacao opcional a conta.

Objetivos financeirosModo fixo ou percentualVinculo opcional a contaUso por planos
Capaz de fazerListar, criar, editar e excluir investimentos com meta, valor atual e status.
LimitacoesNome e modo de contribuicao sao obrigatorios; valores nao podem ser negativos.
PermissoesBearer obrigatorio e filtro por user_id.
Arquivos relacionadosserver/app.ts, server/database.js, src/pages/Investments.tsx.

Visao operacional

Investimentos representam metas ou envelopes de acumulacao e podem ser usados como referencia nos planos. O backend persiste apenas a estrutura do objetivo; nao foi identificada uma rotina automatica de aporte gerando transacoes.

Capaz de fazer

  • Criar investimento com nome, descricao, status, cor e notas.
  • Definir contribuicao fixa ou percentual da renda.
  • Associar investimento a uma conta do usuario.
  • Guardar valor atual e valor alvo.
  • Listar investimentos com paginação opcional.

Nao faz

  • Nao gera transacoes automaticamente no codigo analisado.
  • Nao permite valor atual, valor alvo, aporte fixo ou percentual negativos.
  • Nao aceita incomePercentage fora do intervalo 0 a 100.

Limitacoes

  • contributionMode precisa estar entre os modos suportados.
  • Se a conta associada nao existir, a operacao falha.
  • Nao foi identificado bloqueio de exclusao por referencia em plano; o plano precisa tratar referencia quebrada em camada propria se isso ocorrer.

Endpoints relacionados

Metodo Rota Capacidade Auth Permissao Entrada Resposta Limitacao
GET /api/investments Lista investimentos Bearer Usuario autenticado page, pageSize opcionais { investments } ou lista paginada Sem filtros adicionais identificados
POST /api/investments Cria investimento Bearer Usuario autenticado name, contributionMode, currentAmount etc. 201 + investimento Banco opcional precisa existir
PATCH /api/investments/:id Edita investimento Bearer Usuario autenticado Mesmo payload de create investimento atualizado ID inteiro obrigatorio
DELETE /api/investments/:id Exclui investimento Bearer Usuario autenticado ID 204 Nao encontrado gera 404

Entradas esperadas

Campo Tipo Obrigatorio Regra Observacao
name string Sim Nao vazio Normalizado com trim
contributionMode string Sim Modo suportado pelo backend fixed_amount ou income_percentage
fixedAmount number Condicional Zero ou maior Somente no modo fixo
incomePercentage number Condicional 0 a 100 Somente no modo percentual
currentAmount number Nao Zero ou maior Default 0
targetAmount number Nao Zero ou maior Pode ser nulo

Validacoes

Validacao Quando ocorre Resultado Erro esperado
Nome obrigatorio Create/update Impede investimento sem nome 400 invalid_investment_name
Modo de contribuicao valido Create/update Impede modo desconhecido 400 invalid_investment_contribution_mode
Faixa numerica valida Create/update Bloqueia negativos e percentual fora de 0-100 400 erros especificos por campo
Conta associada existente Create/update Impede conta inexistente 404 bank_connection_not_found
ID inteiro PATCH/DELETE Bloqueia rota malformada 400 invalid_investment_id

Efeitos colaterais

  • Cria, altera ou remove registros em investments.
  • Pode ser referenciado por planos e drafts de planejamento.
  • Frontend invalida listas de investimentos e planos relacionados.

Erros esperados

HTTP Causa Quando Como corrigir
400 Payload invalido Create/update/delete Corrigir modo e valores
404 Investimento ou conta nao encontrados Update/delete Usar recurso valido
401 Bearer ausente Qualquer rota Autenticar
