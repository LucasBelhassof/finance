API de habitacao

Documenta despesas fixas e financiamentos de habitacao. O backend persiste o cadastro e regenera as transacoes derivadas sempre que a despesa e criada ou atualizada.

Gera transacoes derivadasFinanciamento vira parcelasEscopo por usuarioVinculo com categoria e conta
Capaz de fazerListar, criar, editar e excluir despesas de habitacao e financiamentos.
LimitacoesValor deve ser positivo; dueDay entre 1 e 31; financiamento exige pelo menos 2 parcelas.
PermissoesBearer obrigatorio, sempre filtrado por user_id.
Arquivos relacionadosserver/app.ts, server/database.js, src/pages/Housing.tsx.

Visao operacional

Habitacao funciona como cadastro de despesa recorrente ou financiamento. Em vez de apenas guardar configuracao, o backend apaga e recria as transacoes derivadas para manter coerencia com o cadastro atual.

Capaz de fazer

  • Criar despesa de aluguel/condominio/conta recorrente.
  • Criar financiamento com numero definido de parcelas.
  • Relacionar despesa a conta/cartao e categoria.
  • Listar despesas com paginação opcional.
  • Editar e regenerar transacoes derivadas.
  • Excluir a estrutura principal de habitacao.

Nao faz

  • Nao aceita valor zero ou negativo.
  • Nao aceita financiamento com menos de 2 parcelas.
  • Nao preserva historico derivado anterior ao editar; ele e regenerado.

Limitacoes

  • description e expenseType validos sao obrigatorios.
  • bankConnectionId precisa existir e pertencer ao usuario.
  • status=inactive deixa de gerar transacoes.
  • Delete remove o cadastro; comportamento cascata adicional depende do schema e nao foi detalhado aqui.

Endpoints relacionados

Metodo Rota Capacidade Auth Permissao Entrada Resposta Limitacao
GET /api/housing Lista despesas de habitacao Bearer Usuario autenticado page, pageSize opcionais { housing } ou lista paginada Sem filtros adicionais identificados
POST /api/housing Cria despesa/financiamento Bearer Usuario autenticado description, expenseType, amount, dueDay, startDate, bankConnectionId 201 + housing Regenera transacoes imediatamente
PATCH /api/housing/:id Edita e regenera Bearer Usuario autenticado Mesmo payload de create housing atualizado ID inteiro obrigatorio
DELETE /api/housing/:id Remove despesa Bearer Usuario autenticado ID 204 Recurso precisa existir

Entradas esperadas

Campo Tipo Obrigatorio Regra Observacao
description string Sim Trim, nao vazia Tambem base da descricao derivada
expenseType string Sim Precisa existir no conjunto de tipos de habitacao Define se e financiamento
amount number Sim Maior que zero Transacao derivada vira negativa
dueDay number Sim 1 a 31 Usado para gerar datas
startDate data Sim Data valida Base da serie
installmentCount number Condicional Minimo 2 em financiamento Nulo para recorrencia simples

Efeitos colaterais

  • Cria/altera/remove registro em housing.
  • Apaga e recria transacoes derivadas da habitacao.
  • Em financiamento, cria registro em installment_purchases e parcelas em transactions.
  • Pode alimentar notificacoes de vencimento em outros fluxos do sistema.

Erros esperados

HTTP Causa Quando Como corrigir
400 Payload invalido ou ID malformado Create/update/delete Corrigir tipo, valor, data e parcelas
404 Conta ou habitacao nao encontrada Update/delete Usar recurso valido do usuario
401 Bearer ausente Qualquer rota Autenticar
